Expand the ShelfSense Platform Admin into a more comprehensive SaaS owner control center.

Context:
ShelfSense is a multi-tenant inventory SaaS. The customer app runs at app.shelfsenseapp.com and the backend at api.shelfsenseapp.com. A basic Platform Admin module already exists with:
- /admin overview
- /admin/workspaces
- /admin/workspaces/:id
- /admin/users
- /admin/users/:id
- /admin/activity
- AdminAuditLog
- platformRole on User
- Workspace subscription/suspension fields

Current issue:
The Platform Admin is too basic. It only shows simple counts and limited controls. I need it to become a professional SaaS control center where I can manage customers, users, subscriptions, plans/packages, coupons, email templates, announcements, and platform health.

Important:
Do this as Phase 1. Keep it stable. Do not break customer app login/signup/workspace/inventory flows.

PHASE 1 GOALS:
1. Improve Platform Admin dashboard with more actionable SaaS metrics.
2. Add Plans & Packages management.
3. Add Subscription control for workspaces.
4. Add Coupons/Promotions management.
5. Add Email Template management.
6. Add Announcements management.
7. Improve Workspace detail page with more controls and customer health insights.
8. Improve admin navigation/sidebar structure.
9. Keep all admin actions audited.

14. Security Requirements
--------------------------------------------------

Very important:
- All /admin API routes must require platformRole SUPER_ADMIN.
- SUPPORT_ADMIN can be added later, but for now default to SUPER_ADMIN unless already implemented.
- Normal workspace owners must not access admin APIs.
- Do not expose password hashes, reset tokens, verification tokens, JWT secrets, SMTP secrets, DATABASE_URL, or API keys.
- Every create/update/delete/suspend/disable action must create AdminAuditLog.
- Use soft delete/archive, not hard delete.
- Admin actions should include actor user id, action type, target type, target id, metadata, timestamp.
- Avoid impersonation in this phase unless it already exists securely.
