We need to improve the ShelfSense signup and onboarding flow so every new user completes plan selection before entering the main dashboard.

Current objective:
After a user registers, they should be required to select a payment plan as part of the signup/onboarding journey.

The desired flow is:

1. User registers with name, email, password, and business/workspace details.
2. Backend creates:
   - User
   - Workspace
   - Default Location
   - Owner Membership
3. User is then taken to a payment plan selection screen.
4. User selects one of the available plans:
   - Free
   - Basic / Starter
   - Pro / Growth
   - Any existing plans already defined in the application
5. If the user selects the Free plan:
   - No payment is required.
   - Coupon field should not be required.
   - Subscription should be activated immediately as FREE.
   - Signup/onboarding should be marked complete.
6. If the user selects a paid plan:
   - Show billing cycle options if supported: monthly / annual.
   - Show coupon code input.
   - Allow user to apply a coupon before confirming the plan.
   - If coupon gives 100% discount, allow the subscription to be activated without payment.
   - If coupon gives partial discount, calculate and show the discounted payable amount.
   - If no payment gateway is currently integrated, create the subscription in a pending/manual payment state and show a clear message that payment setup is pending.
   - Do not block the technical flow if payment gateway is not implemented yet, but keep subscription status clearly marked.
7. After plan confirmation:
   - Mark signup/onboarding stage as 100% complete.
   - Redirect user to the dashboard.
8. Dashboard must not be accessible until the user has:
   - Registered
   - Created workspace
   - Selected a plan
   - Completed onboarding/signup stage

Important:
Please inspect the existing backend, Prisma schema, frontend routes, auth context, onboarding flow, and any existing subscription/billing models before making changes.
Do not invent duplicate models if subscription, plan, coupon, or onboarding-related models already exist.
Use existing model names and fields wherever possible.

6. Add APIs:
   GET /plans
   - Returns active plans with features, limits, monthly/annual prices, and display metadata.

   POST /subscriptions/preview
   - Input:
     - workspaceId if required from context
     - planId
     - billingCycle
     - couponCode optional
   - Output:
     - plan
     - originalAmount
     - discountAmount
     - payableAmount
     - couponApplied true/false
     - couponMessage
     - canActivateWithoutPayment true/false

   POST /subscriptions/select-plan
   - Input:
     - planId
     - billingCycle
     - couponCode optional
   - Behavior:
     - For FREE plan:
       create/update workspace subscription as FREE/ACTIVE equivalent.
       mark onboarding/signup stage complete.
     - For paid plan with 100% coupon:
       create/update subscription as ACTIVE.
       save coupon redemption.
       mark onboarding/signup stage complete.
     - For paid plan with partial discount and no payment gateway:
       create/update subscription as PENDING_PAYMENT.
       mark plan selection complete but payment pending.
       Keep access behavior configurable. For MVP, allow dashboard access with clear pending payment banner.
     - For paid plan with no coupon and no payment gateway:
       create/update subscription as PENDING_PAYMENT.
       show pending payment status.
       For MVP, allow dashboard access with pending payment banner.
   - Must be workspace-scoped and only workspace OWNER/admin can select/change plan during signup.

   GET /onboarding/status
   - Returns:
     - hasWorkspace
     - hasDefaultLocation
     - hasMembership
     - hasSelectedPlan
     - subscriptionStatus
     - onboardingCompleted
     - nextStep: PLAN_SELECTION / BUSINESS_SETUP / DASHBOARD

7. Backend validation:

   - Fixed coupon should not reduce below zero.
   - Prevent duplicate active subscriptions for the same workspace.
   - Use transactions when selecting a plan and redeeming coupon.
   - Add structured logs for registration and plan selection.
