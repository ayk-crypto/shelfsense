Part C — Background Jobs

1. Add scheduled jobs using a simple scheduler (e.g. node-cron).
2. Jobs required:

   Low stock job:
   - Runs every few hours
   - Finds items below minimum stock
   - Sends alert email if enabled
   - Avoid duplicate spam (do not send repeatedly for same unchanged item)

   Expiry job:
   - Runs every few hours or daily
   - Finds items expiring soon
   - Sends alert if enabled

   Daily digest job:
   - Runs once per day
   - Includes:
     - low stock items
     - expiring soon items
     - expired items
   - Send one summary email per workspace/user

3. Ensure jobs are safe:
   - Workspace scoped
   - Handle errors gracefully
   - Do not crash server

Part D — Anti-Spam Logic

1. Avoid sending duplicate alerts:
   - Track last sent timestamp per item/workspace/type
2. Do not resend same alert too frequently.
3. Only send when:
   - New condition triggered OR
   - Daily digest time reached
